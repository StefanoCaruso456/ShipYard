# Contributing

## Workflow

For Codex work on this repo:

- use one session branch
- do not push directly to `main`
- before closing out a completed task, commit the task work on the current session branch
- push the session branch when the work is done
- open a pull request into `main`, or update the existing open session pull request
- the user reviews and merges
- do not present a repo-changing task as complete until the commit is pushed and the PR exists or is updated
- if a task does not change repository files, say that explicitly in the final response instead of pretending there was a commit

Branch prefix:

```text
codex/
```
