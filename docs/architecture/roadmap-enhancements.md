# Roadmap Enhancements

## Purpose

Document the next product-facing layer of Shipyard after the current runtime core.

These phases are based on a review of the minimum-viable-factory pattern:

- clearer operator workflow
- explicit approvals
- visible progress
- stronger external visibility
- a simpler factory experience on top of a more capable runtime

Shipyard should adopt those strengths without giving up its typed control plane, bounded tools, or
runtime-first architecture.

## Enhancement Sequence

1. Phase 15: Operator Workflow Foundation
2. Phase 16: Human Approval Gates
3. Phase 17: Orchestrator Artifacts and Structured Decomposition
4. Phase 18: External Record Sync
5. Phase 19: Factory Mode
6. Phase 20: Merge and Conflict Governance
7. Phase 21: Delivery Summary and Operator Evaluation

## Guiding Rule

These enhancements are additive.

They should:

- build on the typed runtime
- make operator workflow clearer
- improve reviewability and safety
- avoid turning markdown memory or external ticket systems into the source of truth

## Expected Result

If these phases are implemented well, Shipyard should keep its stronger technical core while gaining
the operational clarity that makes a software factory easier to trust and operate.
